Learning Salesforce Programming

My experiences with it so far. December, 2019

Persistence! At university I had covered front-end web development, object relationships in databases, MVC structure and dabbled in PHP, but had never done back-end coding like I do now with Lightning Aura components.

At the beginning of learning to program I felt I'd never 'make it' because there were so many languages to learn, and so muany admin out-the-box features to get to grips with. It was overwhelming at times, but then through the support of excellent and experienced developers I learnt everyone felt that way at some stage, and even with decades of experience, many felt there was always something new to learn to sharpen their skills.

This relaxed me, and I took a deeper dive into programming, getting to grips with:

  • The flow between the component, controller, helper and apex class
  • How to use the Community Builder
  • Writing test classes
  • Lightning data tables and wrapper classes
  • Page layouts and record types, field-level security, global picklists
  • Workflow rules

It has been a journey, but one I've thoroughly enjoyed from the frustrations to the joys. There's something about programming that brings such pride and satisfaction when you create a piece of code that is going to help the project succeed.
