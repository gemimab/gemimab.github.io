Learning Salesforce Programming

My experiences with it so far. September, 2020

The brightside of 2020. This year went fast. In a blink of an eye it became September, but despite the many challenges 2020 brought, for technical growth I'm at my proudest.

Almost a year ago I felt almost overwhelmed by the mountain infront of me which was Salesforce dev. With so many new languages at once, so many syntax variations, it was hard getting my feet off the ground.

But, through continued support from team members and my mentor, everything began to click and no longer seem impossible. There is still so much to learn, and with Salesforce it will always be that way as it is such a vast CRM, but that is something that excites me.

  • I chose to take on the challenge of creating my first LWC, which was a steep learning curve, as it wasn't being used by anyone on the dev team yet
  • I've learnt more about apex triggers, events, maps, aura binding and wrapper classes

Persistence! At university I had covered front-end web development, object relationships in databases, MVC structure and dabbled in PHP, but had never done back-end coding like I do now with Lightning Aura components.

At the beginning of learning to program I felt I'd never 'make it' because there were so many languages to learn, and so many admin out-the-box features to get to grips with. It was overwhelming at times, but through talking with experienced developers I realised my feelings were common, and to be a good developer, you need to be constantly learning.

This relaxed me, and I took a deeper dive into programming, getting to grips with:

  • The flow between the component, controller, helper and apex class
  • How to use the Community Builder
  • Writing test classes
  • Salesforce Admin features

It has been a journey, but one I've thoroughly enjoyed from the frustrations to the joys. There's something about programming that brings such pride and satisfaction when you create a piece of code that is going to help the project succeed.
